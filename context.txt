⏺ 좋은 질문입니다. Godot에서 미니게임 통합 방식을 정리해보면:

  미니게임 통합 아키텍처

  ┌─────────────────────────────────────────────────────────┐
  │                    Parent Game                          │
  │  ┌─────────────┐  ┌─────────────┐  ┌─────────────┐     │
  │  │ SaveSystem  │  │ PlayerData  │  │ Inventory   │     │
  │  └──────┬──────┘  └──────┬──────┘  └──────┬──────┘     │
  │         └────────────────┼────────────────┘            │
  │                          ▼                              │
  │              ┌───────────────────────┐                  │
  │              │   MiniGameManager     │                  │
  │              │  - start_minigame()   │                  │
  │              │  - on_minigame_end()  │                  │
  │              └───────────┬───────────┘                  │
  └──────────────────────────┼──────────────────────────────┘
                             │ 입력: 캐릭터, 난이도, 보너스
                             │ 출력: 점수, 생존시간, 보상
                             ▼
  ┌─────────────────────────────────────────────────────────┐
  │                  Mini Game (test-game)                  │
  │  ┌─────────────────────────────────────────────────┐   │
  │  │              MiniGameInterface                   │   │
  │  │  - setup(config: Dictionary)                    │   │
  │  │  - get_result() -> MiniGameResult               │   │
  │  │  - signal game_ended(result)                    │   │
  │  └─────────────────────────────────────────────────┘   │
  └─────────────────────────────────────────────────────────┘

  Godot에서 연결 방식

  | 방식                         | 장점                | 단점      |
  |------------------------------|---------------------|-----------|
  | 씬 전환                      | 간단, 메모리 분리   | 로딩 필요 |
  | SubViewport                  | 동시 실행 가능, PIP | 성능 비용 |
  | 별도 프로젝트 → PCK 익스포트 | 완전 독립           | 빌드 복잡 |

  필요한 인터페이스

  # 미니게임 입력
  class_name MiniGameConfig
  var character_id: String      # 외부 캐릭터 연동
  var difficulty: int           # 난이도 (1-5)
  var time_limit: float         # 제한 시간 (0=무제한)
  var bonus_stats: Dictionary   # 외부 보너스 적용

  # 미니게임 출력
  class_name MiniGameResult
  var score: int                # 점수
  var survival_time: float      # 생존 시간
  var kills: int                # 처치 수
  var items_collected: Array    # 획득 아이템 ID
  var success: bool             # 성공/실패

  선택지

  어떤 방향으로 갈지에 따라 구현이 달라집니다:

  1. 같은 프로젝트 내 씬 - 부모 게임과 미니게임이 하나의 Godot 프로젝트
  2. 별도 프로젝트 (PCK 모듈) - 미니게임을 독립 빌드 후 임포트
  3. SubViewport 내장 - 미니게임을 창 안의 창처럼 실행

> ㄴㄴ 아직 부모 게임이 없어. 같은 엔진으로 하는게낫겠지? 

⏺ 네, 같은 Godot 엔진이면 훨씬 간단합니다.

  추천 구조

  godot-games/
  ├── main-game/           # 부모 게임 (나중에 생성)
  │   ├── project.godot
  │   └── ...
  │
  ├── test-game/           # 미니게임 (현재)
  │   ├── project.godot
  │   └── ...
  │
  └── _shared/             # 공유 모듈 (이미 있음!)
      └── scripts/

  _shared/ 폴더가 이미 있어서 공유 구조가 갖춰져 있습니다.

  지금 해두면 좋은 것

  미니게임 입출력 인터페이스를 먼저 만들어두면, 나중에 부모 게임 만들 때 바로 연결 가능:

  # 미니게임 시작 시
  MiniGameInterface.start({
      "character": "warrior",
      "difficulty": 2,
      "time_limit": 600  # 10분
  })

  # 미니게임 종료 시 → 부모 게임에 결과 반환
  signal game_ended(result: MiniGameResult)
  # result = { score: 1500, kills: 47, survival_time: 423.5, success: true }

  다음 단계

  1. MiniGameInterface - 시작/종료 인터페이스 추가
  2. MiniGameResult - 결과 데이터 구조 정의
  3. GameManager 연동 - 기존 통계를 Result로 패키징

  이 인터페이스 시스템 구현할까요? 기존 GameManager에 이미 total_kills, game_time 같은 통계가 있어서 연결만 하면 됩니다.
